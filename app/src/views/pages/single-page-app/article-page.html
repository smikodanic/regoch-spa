Page

class Page extends DataRg

This class will load and render views in the data-rg-view and includes in the data-rg-inc elements.
- Views are the HTML files located in the /src/views/ directory and loaded only when it is called by the controller. Which controller i.e. view will be taken depends on the URL. The views are loaded in the element with data-rg-view attribute. The possible load destinations are: inner, outer, sibling, prepend and append.
- Includes are HTML files located in the /src/views/inc/ directory and will be rendered every time regardless to the controller and URL. The inc files are loaded in the elements with the data-rg-inc attribute. The possible load destinations are: inner, outer, sibling, prepend and append.

Properties

Property Description Type Default
baseURIhost Current browser's base URI, for example: http://localhost:4400 string
httpClient HTTP client, instance of the HTTPClient class object

Methods

  • async loadInc (delIncgens:boolean): Promise[void]

    Load inc HTML file into the element with the data-rg-inc="file.html [@@ inner|outer|sibling] [@@ cssSelector]" attribute. This method is called recursively if the "data-rg-inc" elements are nested.
    The render process:
    1) Select all elements which has data-rg-inc but not data-rg-cin.
    2) Put data-rg-cin which marks that the data-rg-inc element has beed parsed in the current iteration.
    3) Load the content in the data-rg-inc element as inner, outer, sibling, append or prepend. Every loaded element will have data-rg-incgen attribute to mark elements generated with data-rg.inc.
    4) Add data-rg-cin attribute to the element with the data-rg-inc to mark that the content is loaded and prevent load in the next iteration.
    note) Multiple iterations will haeppen when data-rg-inc elements are nested. To make reset all data-rg-incgen elements will be deleted but only in the first iteration.
    delIncgens - to delete the elements with "data-rg-incgen" atribute (only in the first iteration and has effect only when sibling, append and prepend are used)

Examples

data-rg-inc

The example shows how to use "data-rg-inc" attribute. In the most cases the method loadInc() shouldn't be used in the controller's code because data-rg-inc elements are rendered automatically. 

<header data-rg-inc="header.html">---load header.html in the innerHTML---</header>
<header data-rg-inc="header.html @@  @@ h2 > small">---inner load (part of the HTML file)---</header>
<header data-rg-inc="header.html @@ inner">---inner load---</header>
<header data-rg-inc="header.html @@ outer">---outer load---</header>
<header data-rg-inc="header.html @@ outer @@ b:nth-child(2)">---outer load (part of the HTML file)---</header>
<header data-rg-inc="header.html @@ sibling">---add HTML file content as sibling elements---</header>
<header data-rg-inc="header.html @@ prepend">---prepend before the existing inner elements---</header>
<header data-rg-inc="header.html @@ append">---append after the existing inner elements---</header></p>

Nested data-rg-inc elements

The example shows nested "data-rg-inc" elements. This will cause two iterative calls of loadInc() method. In the first iteration it will load article.html in the main.html. As the article.html has also data-rg-inc element in the second iteration the content.html will be loaded. 

<!-- main.html -->
<section data-rg-inc="article.html">---will load article.html which has another data-tg-inc---</section>

<!-- article.html -->
<div data-rg-inc="content.html"></div>

<!-- content.html -->
