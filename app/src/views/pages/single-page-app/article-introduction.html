Introduction

The Regoch SPA is framework for developing single page, web applications. Simple and inutitive for use and extremly fast.

 The website regoch.org is completely made in the Regoch SPA framework.

Features

  • lightweight application - very small size of the compiled file (~50kB only)
  • JS files compiled with the gulp and browserify (very fast)
  • use CommonJS and write the code just like you are doing that in NodeJS by using require()
  • no typescript, no heavy compiling, no bullshit
  • Model-View-Controller (MVC), intuitive app structure
  • easy to learn and to use
  • due to its small size you can build very fast mobile apps

Installation

npm install -g regoch-spa

Quick Start

Use Regoch-SPA CLI (Command Line Interface) to init the new app. This command will create the files and directory structure with the basic example app. 

rg new appName


After that open /src/app.js and edit it. 

const { App } = require('regoch-spa');

// route definitions
const routesCnf = [
  ['when', '/', 'HomeCtrl'],
  ['when', '/contact', 'ContactCtrl'],
  ['when', '/product/:id', 'ProductCtrl'],
  ['notfound', 'NotfoundCtrl'],
];

// define configuration and constants
app
  .conf('API', apiCnf)
  .const('PI', 3.14)
  .freeze(); // prevent CONST and CONF accidental modification

// inject helper libraries which you want to use in the controller
app.libInject({ Lib1, lib2 });

app
  .controller([HomeCtrl, ContactCtrl, ProductCtrl, NotfoundCtrl]) // inject controllers in the app
  .routes(routesCnf).run(); // add and run the app routes


Playground

To learn more quickly visit our playground examples:
  • Controller Lifecycle Hooks - shows in which order will be executed constructor, prerender, postrender, init, destroy
  • Page::loadInc() - test the loadInc() method with nested data-rg-inc elements
  • Page::loadViews() - test the loadViews() method with isAsync argument
  • DataRg - test the data-rg- elements related to DataRg methods
  • DataRgListeners - test the data-rg- elements related to DataRgListeners methods
