Controller Hooks Test

Click /playground/controller-hooks-same to destroy the controller.

The example shows in which order will be executed controller constructor and the lifecycle hooks. Open the browser console to see the constructor and hook arguments.
  1. constructor(app) - app is the app instance (initialised when app is started i.e. when browser reload the page)
  2. init(trx) - trx is the regoch router transitional variable. It contains the current URL and other injected data. (initialised by the router)
  3. prerender(trx) - trx is the regoch router transitional variable. It contains the current URL and other injected data. (initialised by the router)
  4. render(trx) - trx is the regoch router transitional variable. It contains the current URL and other injected data. (initialised by the router)
  5. postrender(trx) - trx is the regoch router transitional variable. It contains the current URL and other injected data. (initialised by the router)
  6. destroy(pevent) - pevent popstate or pushstate event which caused URL change (initilialised when the route is changed)
